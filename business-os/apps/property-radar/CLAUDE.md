# 不動産物件収集 — アプリ設計書

## 概要
投資基準に合う不動産物件を自動収集・スコアリングし、日次レポートで通知する。

## 全体フロー

```
情報ソース（楽待、健美家、アットホーム等）
  ↓ GAS で定期スクレイピング or RSS取得
物件データ抽出
  ↓ パース＋正規化
投資基準スコアリング
  ↓ 自動スコア算出
  ├→ Aランク（基準合致）→ 即通知
  ├→ Bランク（価格交渉次第）→ サマリに含む
  └→ Cランク以下 → DBに保存のみ
日次レポート
  ↓ LINE or メールで送信
物件DB（スプレッドシート）に蓄積
```

## 投資基準（オーナーが設定画面でカスタマイズ可能）

| 項目 | デフォルト基準 | 重み |
|------|---------------|------|
| 表面利回り | ≥10% | 高 |
| 実質利回り | ≥7% | 高 |
| エリア | 設定リスト内 | 高 |
| 価格帯 | 500万〜5000万 | 中 |
| 築年数 | ≤30年 | 中 |
| 構造 | RC/S造優先 | 低 |
| 駅徒歩 | ≤15分 | 低 |
| 空室率 | ≤20% | 中 |

### スコア計算
- 各項目を0-100でスコア化
- 重み付き平均でトータルスコア算出
- A: 80以上 / B: 60-79 / C: 40-59 / D: 39以下

### Bランク判定（価格交渉候補）
- スコアCだが利回りだけ基準未達の場合
- 「あと○○万円下がれば基準合致」を自動計算
- 売出期間が長い物件（値下げ可能性高い）を優先
